Network Performance Dashboard
=================================

Overview
--------
This project analyzes the YouTube communities dataset from SNAP (Stanford Network Analysis Project) to identify influential contacts and actionable opportunities. It prepares clean relational data, computes network metrics, and serves an interactive Streamlit dashboard focused on business‑friendly insights.

Key features
------------
- Reproducible cleaning of SNAP communities data
- Projection to a co‑membership network with weighted edges
- Node metrics: degree and weighted_degree (optionally PageRank, Betweenness)
- Simple Mode dashboard for non‑technical users and Advanced Mode for analysts
- Explanatory glossary and narrative blocks embedded in the app

Data source
-----------
- SNAP: YouTube networks with ground‑truth communities
- Dataset page: https://snap.stanford.edu/data/com-Youtube.html
- Files used:
  - com-youtube.all.cmty.txt.gz (communities)
  - com-youtube.top5000.cmty.txt.gz (optional)
  - com-youtube.ungraph.txt.gz (optional)

Dashboard guide
---------------
- Simple Mode (default)
  - KPIs: people in network, detected connections, mapped communities
  - Bar chart: top contacts by connection strength (weighted_degree)
  - Plain‑language explanation of what the chart means

- Advanced Mode
  - Overview: KPIs, top nodes, communities treemap, communities percentage (donut)
  - Network & Influence: filtered subgraph of influential nodes with optional PageRank/Betweenness
  - Hierarchies & Segments: dendrogram and size‑based hierarchy
  - Recommendations: prioritization table (downloadable CSV)
  - Methodology: data source, process, and glossary

Concepts (short reference)
--------------------------
- Node: user/account in the network
- Edge: connection created by co‑membership; weight = co‑occurrence frequency
- Weighted_degree (connection strength): degree weighted by tie strength; primary proxy for amplification
- PageRank (optional): importance considering neighbors’ importance
- Betweenness (optional): bridging ability across groups
- Approx. density: connectedness of the network (0..1)

Troubleshooting
---------------
- Missing files warning in the app: run the notebook to regenerate artifacts under data/processed and reports/figures.
- Port in use: change the Streamlit port (e.g., 8503) with --server.port.
- Kernel or permissions issues on macOS: run commands from a local terminal and allow Python incoming connections when prompted.
